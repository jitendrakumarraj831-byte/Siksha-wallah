Section 6
Security Architecture
🛡️ Rate Limiting
Admin login: 5 attempts/5 min. Email verification: 5/15 min. Contact form: 5/10 min. All per IP address.
🍪 Secure Admin Sessions
httpOnly, sameSite=lax cookies. HMAC-SHA256 signed tokens. No Firebase Auth for admin — completely separate security model.
🗄️ Firestore Security Rules
Students can only read/write their own data. Guests can submit inquiries but cannot read data. Admin uses server-side SDK (bypasses rules).
✅ Input Validation
Zod schemas validate all form inputs on both client and server. No raw unsanitised data enters the database.

PWA — Progressive Web App
📲 Installable on Phone
"Add to Home Screen" on Android/Chrome. App runs in standalone mode (no browser UI bar) — looks and feels like a native app.
⚡ Service Worker
Network-first caching. Static assets (CSS, JS, fonts, images) cached locally. Offline fallback to homepage. Fast repeat visits.
🎨 Web App Manifest
Custom app name, theme color (#003f9f), 192x192 and 512x512 icons. Splashscreen-like launch experience.
📴 Offline Support
Homepage and key static pages available offline. API responses excluded from cache to ensure fresh data.
